# Ranger — гайд и шпаргалка

Конфиг: [`config/ranger/`](../config/ranger/). [Ranger](https://github.com/ranger/ranger) — консольный файловый менеджер с vim-клавишами и трёхколоночным видом.

## Иерархия (что есть что)

```mermaid
flowchart LR
  P[Колонка 1 — родитель]
  C[Колонка 2 — текущая папка]
  V[Колонка 3 — превью]
  P --- C --- V
```

| Уровень | В ranger | На экране | Аналог |
|---------|----------|-----------|--------|
| Вкладка | **Tab** | полоска вкладок сверху/снизу | tmux **window** |
| Папка | **Directory** | средняя колонка (фокус) | cwd в shell |
| Файл | **File** | строка в списке | путь к файлу |
| Закладка | **Bookmark** | `m` + `'` | быстрый jump |

**Важно:** ranger — это **навигация по ФС**, не редактор. Файлы открываются через **rifle** (по умолчанию — **nvim**).

## Установка

```bash
# Debian/Ubuntu/WSL
sudo apt install ranger

# или из PyPI
pipx install ranger-fm
```

Конфиг из репо:

```bash
cd ~/develop/project/dotnetsh   # путь к репо
stow -t ~/.config config        # → ~/.config/ranger/
```

Запуск: `ranger` или `ranger /path/to/dir`.

## Интерфейс

Три колонки слева направо:

1. **Родительская** директория — быстрый выход на уровень вверх (`h`).
2. **Текущая** — здесь основная работа, vim-навигация.
3. **Превью** — содержимое выделенного файла/папки (текст, метаданные).

Статусбар внизу: путь, git-статус (если `vcs_aware`), количество файлов, подсказки.

## Навигация (vim-клавиши)

Ranger по умолчанию использует **hjkl** — как в vim/neovim.

| Клавиши | Действие |
|---------|----------|
| `h` | На уровень вверх (родитель) |
| `l` / `Enter` | Войти в папку / открыть файл |
| `j` / `k` | Вниз / вверх по списку |
| `gg` / `G` | В начало / конец списка |
| `Ctrl+f` / `Ctrl+b` | Страница вниз / вверх |
| `[` / `]` | Родитель / дочерняя папка в истории |
| `zh` | Показать/скрыть dotfiles |
| `H` | История посещённых директорий |
| `r` | Обновить содержимое |

### Кастомные переходы (в конфиге)

| Клавиши | Действие |
|---------|----------|
| `gh` | Домашняя директория (`~`) |
| `uu` | На уровень вверх (`cd ..`) |

## fzf — поиск файлов

Команда `fzf_select` из [`commands.py`](../config/ranger/commands.py): рекурсивный fuzzy-поиск от **текущей** папки.

| Клавиши | Действие |
|---------|----------|
| `Ctrl+f` | Найти файл или папку → перейти/выделить |
| `gf` | То же |
| `2` + `Ctrl+f` | Только **директории** (квантификатор ranger) |

Поведение:

- Если выбрана **папка** — ranger переходит в неё (`cd`).
- Если **файл** — выделяет в текущем списке.
- Использует `fd`/`fdfind`, если установлен; иначе `find`.
- Скрытые файлы учитываются, если включены через `zh`.

В fzf: **Enter** — выбрать, **Esc** — отмена. Превью — первые 100 строк (`cat`).

> **Зависимости:** `fzf` обязателен. Для скорости — `fd` (`sudo apt install fd-find`, бинарь `fdfind`).

## Открытие файлов (rifle)

Файл `Enter` или `l` запускает **rifle** — лаунчер ranger.

| Тип | Программа |
|-----|-----------|
| Код, текст, конфиги (`.cs`, `.sln`, `.go`, `.lua`, `.md`, …) | **nvim** |
| По SSH (fallback) | **vim** |
| Картинки | feh (если есть) |
| PDF | evince |
| Видео | mpv |
| Остальное | `xdg-open` |

Редактор по умолчанию: `set editor nvim` в [`rc.conf`](../config/ranger/rc.conf).

Открыть **снаружи** ranger (в фоне): `Shift+Enter` (`r` + shift в некоторых версиях — см. `?` help).

## Консоль и shell

| Клавиши | Действие |
|---------|----------|
| `:` | Консоль ranger (команды, `:help`) |
| `` ` `` | Shell в текущей директории (кастом) |
| `S` | Shell (стандартный) |
| `Q` / `q` | Выход |

```text

```

## Интеграция с dotfiles

| Инструмент | Как связан |
|------------|------------|
| **nvim** | rifle открывает код в nvim; `$EDITOR=nvim` в zsh |
| **fzf** | `Ctrl+f` — поиск; в zsh fzf тоже есть (`~/.fzf.zsh`) |
| **tmux** | ranger в pane; `C-hjkl` navigator работает после выхода из ranger |
| **Alacritty** | Nord-подсветка в rc.conf созвучна терминалу |

### Выйти и остаться в папке (опционально)

Ranger умеет менять cwd shell при выходе. В zsh можно добавить в `~/.zshrc.local`:

```bash
ranger() {
  local tmp="${TMPDIR:-/tmp}/ranger-cd-$RANDOM"
  command ranger "$@" --choosedir="$tmp"
  if [[ -f "$tmp" ]]; then
    cd -- "$(cat "$tmp")" || return
    rm -f "$tmp"
  fi
}
```

Тогда после `q` shell окажется в последней папке ranger.

## Типичный workflow

```text
1. ranger ~/develop/project
2. j/k — листать, l — войти в репо
3. Ctrl+f — найти Program.cs → Enter
4. Enter — открыть в nvim (внутри ranger или новый процесс)
5. q — выход обратно в shell
```

С tmux:

```text
tmux pane: ranger | nvim
  → в ranger: Ctrl+f, выбрать файл
  → Enter → nvim в том же TTY
  → или yp → путь в буфере → вставить в другую pane
```

## Частые вопросы

| Вопрос | Ответ |
|--------|--------|
| `Ctrl+f` не работает | Проверь `which fzf`; перезапусти ranger |
| fzf пустой / медленный | Поставь `fd-find`; включи скрытые `zh` если нужно |
| Файл открывается не в nvim | `:open_with nvim` или проверь `rifle.conf` |
| Нет превью картинок | В минимальном конфиге превью — текст; для картинок нужен `scope.sh` |
| Конфликт `Ctrl+f` | В ranger это fzf; в shell fzf — `Ctrl+t` / `Ctrl+r` (стандарт fzf) |
| Где конфиг | `~/.config/ranger/{rc.conf,rifle.conf,commands.py}` |
